# Agent_熵管理规范

---

## 1. 核心原则

> **Agent 持续运行，系统一定会逐渐"变脏"**

prompt 越来越长、rules 越来越旧、工具定义越来越多但没人维护。

---

## 2. 熵的来源

| 来源 | 表现 | 影响 |
|------|------|------|
| Prompt 膨胀 | 上下文越来越长 | 性能下降、推理变慢 |
| Rules 堆积 | 规则越来越多 | 冲突、难以维护 |
| 工具过载 | 工具越来越多 | 发现困难、选择困难 |
| 上下文泄漏 | 旧上下文未清理 | 记忆混乱 |
| 知识陈旧 | 规则/配置过期 | 行为不一致 |

---

## 3. 熵管理策略

### 3.1 上下文裁剪

| 策略 | 说明 | 触发条件 |
|------|------|----------|
| 消息裁剪 | 删除旧的对话消息 | 超过 max_tokens 阈值 |
| 消息压缩 | 压缩历史消息 | 对话长度超过阈值 |
| 关键上下文保留 | 保留重要上下文 | 裁剪时优先保留 |

### 3.2 Rules 定期清理

| 规则类型 | 生命周期 | 审查频率 |
|----------|----------|----------|
| 临时规则 | 7 天后自动过期 | - |
| 长期规则 | 90 天审查一次 | 每季度 |
| 永久规则 | 180 天审查一次 | 每半年 |

### 3.3 工具清理

| 策略 | 说明 |
|------|------|
| 低使用率工具标记 | 使用率低于阈值 (1%) 的工具标记审查 |
| 旧版本工具标记 | 超过 2 个版本的工具标记升级 |
| 通知维护者 | 低使用率工具自动通知维护者 |

---

## 4. 熵指标

| 指标 | 阈值 | 告警级别 |
|------|------|----------|
| 上下文长度 | > 10000 tokens | warning |
| 有效 Rules 数 | > 100 | warning |
| 有效工具数 | > 50 | warning |
| 低使用率工具占比 | > 20% | info |

---
