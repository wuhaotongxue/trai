# PostgreSQL 主键自增方案详解

---

## 概述

PostgreSQL 中有三种等效的主键自增方案，本文档详细对比它们的区别和联系。

---

## 三种方案对比

### 1. Serial（序列）— 旧式写法

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(64)
);
```

PostgreSQL 实际执行了以下步骤：

```sql
-- 1. 创建序列
CREATE SEQUENCE users_id_seq;

-- 2. 创建表，字段类型为 INTEGER
CREATE TABLE users (
    id INTEGER NOT NULL DEFAULT nextval('users_id_seq'),
    username VARCHAR(64),
    PRIMARY KEY (id)
);

-- 3. 设置序列属主
ALTER SEQUENCE users_id_seq OWNED BY users.id;
```

**特点**：
- 自动创建序列 `users_id_seq`
- 默认值使用 `nextval()` 从序列取值
- **字段类型是 INTEGER（32 位）**，最大约 21 亿

---

### 2. BigInt Identity — 新式写法（SQLAlchemy 推荐）

```sql
CREATE TABLE users (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    username VARCHAR(64)
);
```

**特点**：
- **字段类型是 BIGINT（64 位）**，最大约 9.2 京（比 Serial 大 2000 倍）
- 不需要手动创建序列，数据库自动管理
- 标准 SQL 语法（SQL:2003）
- 可以显式指定起始值和步长

```sql
-- 指定起始值为 1000，步长为 1
id BIGINT GENERATED BY DEFAULT AS IDENTITY (START WITH 1000 INCREMENT BY 1) PRIMARY KEY
```

---

### 3. autoincrement=True（SQLAlchemy 写法）

在 SQLAlchemy 中，`autoincrement=True` 会根据数据库类型生成对应的 DDL：

```python
id: Mapped[int] = mapped_column(Integer, primary_key=True, autoincrement=True)
```

在 **PostgreSQL** 上会被渲染为：

```sql
id INTEGER NOT NULL DEFAULT nextval('table_id_seq'::regclass)
```

即等同于 **Serial** 的写法。

---

## 三者区别总结

| 特性 | Serial | BigInt Identity | autoincrement=True |
|------|--------|-----------------|-------------------|
| 字段类型 | INTEGER (32 位) | BIGINT (64 位) | INTEGER (32 位) |
| 最大值 | 2,147,483,647 | 9,223,372,036,854,775,807 | 2,147,483,647 |
| 序列名称 | 手动指定 | 自动生成 | 自动生成 |
| SQL 标准 | PostgreSQL 特有 | SQL:2003 标准 | PostgreSQL 特有 DDL |
| 显式控制步长 | 否 | 是 | 否 |
| 适用场景 | 小型项目快速原型 | 生产环境推荐 | SQLAlchemy ORM 映射 |

---

## 为什么推荐 BigInt Identity

### 1. 数据容量���距巨大

```
Serial 最大值：      2,147,483,647  （约 21 亿）
BigInt 最大值： 9,223,372,036,854,775,807  （约 9.2 京）
```

对于高并发系统，INTEGER 可能在项目生命周期内耗尽，BIGINT 几乎不可能。

### 2. 标准兼容

BigInt Identity 是 ISO SQL 标准写法，便于：
- 数据库迁移（Oracle、MySQL 8+、SQL Server 都支持）
- 团队统一规范
- 文档和代码审查

### 3. SQLAlchemy 中的正确写法

**❌ 不推荐（会生成 Serial）**：
```python
id: Mapped[int] = mapped_column(Integer, primary_key=True, autoincrement=True)
```

**✅ 推荐（生成 BigInt Identity）**：
```python
from sqlalchemy import BigInteger

id: Mapped[int] = mapped_column(BigInteger, primary_key=True, autoincrement=True)
```

或使用 SQLAlchemy 的 Identity 生成器：

```python
from sqlalchemy import BigInteger

id: Mapped[int] = mapped_column(
    BigInteger,
    primary_key=True,
    autoincrement="auto"  # 尽量用数据库的 IDENTITY 特性
)
```

## 实际验证

可以通过以下 SQL 检查表使用的自增方案：

```sql
-- 查看表的主键列类型
SELECT
    c.relname AS table_name,
    a.attname AS column_name,
    format_type(a.atttypid, a.atttypmod) AS data_type,
    a.attnotnull AS not_null,
    pg_get_expr(ad.adbin, ad.adrelid) AS default_value
FROM pg_attribute a
JOIN pg_class c ON c.oid = a.attrelid
JOIN pg_namespace n ON n.oid = c.relnamespace
LEFT JOIN pg_attrdef ad ON ad.adrelid = c.oid AND ad.adnum = a.attnum
WHERE c.relkind = 'r'
  AND n.nspname = 'public'
  AND a.attnum > 0
  AND a.attisdropped = false
ORDER BY c.relname, a.attnum;
```

## TRAI 项目规范

根据 TRAI 后端数据库规范：

| 规则 | 说明 |
|------|------|
| 主键类型 | BigInt Identity |
| SQLAlchemy 写法 | `BigInteger` + `autoincrement="auto"` |
| 禁止 | Serial / `autoincrement=True` + `Integer` |
