## [20:05] ❌ 想当然：企业微信模板卡片参数错误

**严重程度**：中等 | **状态**：✅ 已修复

### 问题摘要

联系消息通知到企业微信时，模板卡片发送失败，多次调试后发现是企业微信 API 文档理解不准确。

### 前因

代码中直接写了企业微信模板卡片参数，未仔细阅读官方 API 文档，导致参数格式和必填字段错误。

### 经过

1. 测试联系表单提交功能
2. 飞书通知发送成功
3. 企业微信发送失败，错误：`Template_Card horizontal_content_list Invalid Size`
4. 查文档发现限制最多 **6 个**字段（不是 3 个）
5. 修改为 6 个字段，但仍然报错 `Missing keyname`
6. 发现字段名应为 `keyname`/`value`（不是 `title`/`desc`）
7. 修改字段名后，仍报 `invalid Request Parameter`
8. 最终查阅官方文档发现：**`text_notice` 类型必须包含 `card_action` 字段**
9. 添加 `card_action={"type":1,"url":"..."}` 后发送成功

### 根因

|| 维度 | 分析 |
|------|------|
| 直接原因 | 企业微信 API 要求：1. `card_type` 必须是官方指定的类型名；2. `text_notice` 必须有 `card_action` |
| 根本原因 | 未先查阅官方 API 文档，想当然地实现 |

### 改进措施

1. ✅ 将 horizontal_content_list 精简为 3 个核心字段（姓名、电话、咨询类型）
2. ✅ 修正字段名为 `keyname`/`value`
3. ✅ 添加 `card_action` 字段（必填）
4. ✅ 使用 `text_notice` 类型而非 `normal_news`
