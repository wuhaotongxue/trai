## [15:00] ❌ 记忆偏差：Grep 工具搜索大文件失败

**严重程度**：中 | **状态**：✅ 已修复

### 问题摘要
使用 Grep 工具搜索 `.env` 大文件（4564行）时，返回 "No matches found"，但文件中实际存在匹配内容。

### 前因
- `.env` 文件是一个超大的配置文件（4564行）
- Grep 工具在大文件上可能存在性能或匹配问题
- 想当然地认为 "No matches found" 就代表真的没有

### 经过
1. 用户询问企业微信和飞书配置
2. 使用 `Grep` 搜索 `NOTIFY_`、`FEISHU`、`WECOM` 等关键词
3. Grep 返回 "No matches found"
4. 又尝试搜索 `webhook` 仍返回空
5. 最后用 `Shell` + `Select-String` 命令找到了所有配置
6. 意识到 Grep 工具在大文件上可能有问题

### 根因

| 维度 | 分析 |
|------|------|
| 直接原因 | Grep 工具对超大文件搜索失效 |
| 根本原因 | 工具局限性认知不足，盲目信任工具输出 |
| 触发条件 | 搜索 `.env` 这类超大配置文件 |

### 改进措施

1. **对搜索结果持怀疑态度**：当搜索结果与预期不符时，主动用 Shell 命令验证
2. **记录工具局限性**：在 skill 中标注 Grep 对大文件的限制
3. **交叉验证**：重要配置搜索至少用两种方式验证

### 教训

> 工具返回 "No matches" 不代表真的没有，可能是工具本身的问题。
> 对于关键配置搜索，应该交叉验证。
