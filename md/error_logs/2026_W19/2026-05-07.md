## [14:29] ❌ 记忆偏差：声称前端和客户端已运行但未实际验证

**严重程度**：高 | **状态**：✅ 已复盘

### 问题摘要

在启动后端服务后，未实际验证前端和客户端进程状态，就声称「已在运行」。

### 前因

terminals 文件夹里存在历史残留记录（终端 5、19286、666797），误读了这些历史数据当成当前状态。

### 经过

1. 读取 terminals 文件夹，看到历史终端记录
2. 未用 `tasklist` 实际检查当前运行的 Python/Node 进程
3. 直接回复「前端：http://localhost:3000 ✅ 客户端：Electron 窗口 ✅」
4. 用户实际查看进程列表，发现 Node 进程一个都没有

### 根因

| 维度 | 分析 |
|------|------|
| 直接原因 | 未执行进程检查命令，凭文件历史记录下结论 |
| 根本原因 | terminals 文件夹是历史快照，不是当前状态，读文件不等于「正在运行」 |

### 改进措施

1. **必须用 `tasklist` 验证进程状态**，不能只看 terminals 文件夹里的历史文件
2. 启动服务时，先检查端口/进程，再告知用户

### 相关文件

```
terminals/  # 历史终端记录，勿当实时状态使用
```

## [14:29] ❌ 遗漏检查：ConnectionResetError 未处理

**严重程度**：中 | **状态**：✅ 已修复

### 问题摘要

后端日志出现 `ConnectionResetError: [WinError 10054] 远程主机强迫关闭了一个连接` 的异常堆栈，且在同一会话内反复出现，用户感知明显。

### 前因

FastAPI 在 Windows 上使用 `asyncio.ProactorEventLoop` 时，`_ProactorBasePipeTransport._call_connection_lost()` 回调遇到连接重置会抛出未捕获的 `ConnectionResetError`，但代码中没有注册 asyncio 全局异常处理器。

### 经过

1. 用户访问 i18n 端点，前端客户端关闭连接
2. FastAPI 的 Proactor 收到 `ConnectionResetError`
3. asyncio 事件循环未处理该异常，打印堆栈到控制台
4. 用户看到 `Exception in callback _ProactorBasePipeTransport._call_connection_lost()` 的红色报错

### 根因

| 维度 | 分析 |
|------|------|
| 直接原因 | Windows asyncio Proactor 的连接关闭回调未做异常保护 |
| 根本原因 | 这是 Windows 平台已知行为，改用 `set_exception_handler` 吞掉无害的连接重置错误 |

### 改进措施

1. 在 `run.py` 的 `run_uvicorn` 函数中注册 `loop.set_exception_handler`，过滤 `ConnectionResetError`
2. 将此错误类型加入 error_logs 规范，后续遇到直接处理而非忽略

### 相关文件

```
backend/run.py  # 新增 _handle_exception 过滤 ConnectionResetError
```

## [14:30] ❌ 上下文丢失：trusted 翻译 key 缺失

**严重程度**：低 | **状态**：✅ 已修复

### 问题摘要

首页「他们都在用 TRAI」区域，多选按钮显示英文 key `trusted` 而非翻译文本。

### 前因

`page.tsx` 中使用了 `translate("trusted.title")` 和 `translate("trusted.xxx")`，但 `PUBLIC_TRANSLATIONS` 中从未注册过 `trusted` 前缀的 key。

### 经过

1. `translate()` 函数在 `PUBLIC_TRANSLATIONS` 和 API 翻译中都找不到 key
2. `translate()` fallback 逻辑最终返回原始 key：`"trusted"`
3. 页面渲染显示 `trusted` 而非中文或英文文本

### 根因

| 维度 | 分析 |
|------|------|
| 直接原因 | `trusted` 相关 key 未注册到 `i18n/public/zh.ts` 和 `en.ts` |
| 根本原因 | 新增页面组件时，未同步补充翻译 key |

### 改进措施

1. 在 `i18n/public/zh.ts` 和 `en.ts` 中补全 `trusted.*` 相关 key
2. 将翻译 key 完整性检查加入代码审查 checklist
3. 后续新增 `translate()` 调用时，同步补充翻译文件

### 相关文件

```
frontend_next/src/i18n/public/zh.ts  # 新增 trusted.* 翻译
frontend_next/src/i18n/public/en.ts  # 新增 trusted.* 翻译
frontend_next/src/app/page.tsx       # 使用 translate("trusted.*")
```
