# Backend_API_开发规范

---

## 1. 核心原则

- **中文标点禁令**：严禁在代码和注释中使用中文全角标点。
- **DDD 架构**：严格遵守 API -> Application -> Domain/Infrastructure 的分层调用顺序。
- **类封装**：所有功能必须封装在类中，严禁顶层孤立函数。

---

## 2. 核心 Skills

- **禁止硬编码绝对路径**：严禁在代码中写死如 `/home/qyjgylc_whf/code/trai/` 等本地绝对路径。所有路径必须基于项目根目录动态计算（如使用 `Path(__file__).resolve().parents[n]`）或从环境变量/配置文件中读取。
- **强制自查导入符号**：在使用 `Path`, `datetime`, `uuid` 等标准库符号前，必须确保文件顶部已正确 `import`。严禁因漏写导入导致 `NameError`。
- **提示词工程优化**：在音频/图像生成接口中，应具备基础的 Prompt 增强逻辑。图像生成必须接入 `PromptOptimizer` 通过 LLM (DeepSeek) 将用户简单的描述润色为专业、丰富的绘画指令，提升画面美感。
- **多媒体推送卡片规范**：向飞书/企业微信推送生成结果时，禁止仅发送纯文本链接。必须使用“图文卡片”（企微 News 类型/飞书 Post 类型）形式，确保图片缩略图能直接在消息列表中展示，提升用户预览体验。
- **节日仪式感规范**：所有向飞书/企业微信推送的媒体通知，必须优先调用 `FestivalManager` 检查当日是否有节日。若有节日，必须通过 `DeepSeek` 动态生成不重复的文案进行推送，增加系统的人文关怀。
- **推送多样化红线**：禁止发送内容完全重复的推送文案。对于节日推送，必须结合“节日背景+作品内容+随机创意”通过 LLM 生成动态内容，确保用户每次收到的祝福都是独一无二的。
- **多模态完整链路规范**：在音乐/视频生成等复杂任务中，应优先构建完整体验，如音乐生成必须包含“音频+动态歌词+封面图”的全套输出，提升产品的专业度与用户体验。
- **环境自动修复机制**：对于本地模型依赖（如 Sambert 依赖的 `kantts`），应在代码中实现检测与自动 `pip install` 的修复逻辑，最大限度保证数字人等 AI 服务的可用性。
- **禁止硬编码密码**：所有数据库密码、API 密钥必须通过环境变量读取，禁止写死在代码中。
