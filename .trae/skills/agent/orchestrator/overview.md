# Agent_编排器总览

---

## 1. 核心定位

> Orchestrator = 多 Agent 系统的中央调度器，是整个 Agent 协作网络的入口和出口。

---

## 2. 编排器类型

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| **Coordinator** | 任务协调者，负责分解和分发 | 复杂任务、多步骤 |
| **Director** | 导演角色，负责编排执行顺序 | 有强依赖关系的任务 |
| **Supervisor** | 监督者，负责监控和纠错 | 长周期任务、容错要求高 |
| **Router** | 路由器，负责路由到合适 Agent | 简单任务、快速响应 |

---

## 3. 任务处理流程

```
[用户请求]
    |
    v
+------------------+
|  Intent Parser   |  意图解析
+------------------+
    |
    v
+------------------+
| Complexity Judge |  复杂度评估
+------------------+
    |
    +---> 简单 (<=4) ---> [单 Agent 执行]
    |
    +---> 复杂 (>=5) ---> [Coordinator]
    |                         |
    |    +---------------------+---------------------+
    |    |                     |                     |
    |    v                     v                     v
    | [Agent A]            [Agent B]            [Agent C]
    |    |                     |                     |
    |    +---------------------+---------------------+
    |                         |
    |                         v
    |               [Result Aggregator]
    |                         |
    |                         v
    +---------------------> [Final Output]
```

---

## 4. 与各模块的关系

| 模块 | 关系 | 说明 |
|------|------|------|
| `interface` | 依赖 | 使用标准化输入输出接口 |
| `protocol` | 依赖 | 通过消息协议与 Agent 通信 |
| `context` | 依赖 | 管理多 Agent 共享上下文 |
| `security` | 约束 | 遵守 RBAC 和权限规则 |
| `monitoring` | 上报 | 上报执行指标 |
| `state` | 依赖 | 管理会话和检查点 |

---

## 5. 支持的任务类型

| 任务类型 | 说明 | 典型流程 |
|----------|------|----------|
| **会议助手** | 一句话触发多 Agent 协作 | 转录 -> 分析 -> 摘要 -> 发送 |
| **报告生成** | 综合多数据源生成报告 | Git 分析 -> 数据收集 -> 撰写 -> 导出 |
| **智能客服** | 多轮对话 + 工具调用 | 意图识别 -> 知识库查询 -> 回答 |
| **内容审核** | 多维度审核内容 | 文字审核 -> 图片审核 -> 结果聚合 |

## 6. 禁止事项
